Software Platforms and Databases

Our software stack

  • HTML, CSS, and Javascript (ReactJS) for frontend
    • ReactJS is a front end javascript framework that is really useful for rendering dynamic html widgets. It makes html generation much easier by allowing html to be written literally rather than as Javascript objects. This will speed up development time significantly
  • Flask as a backend web framework
    • Flask is a micro web framework written in Python that makes a lot of things a lot easier in terms of backend web serving. It makes it very easy to define URL routes, messaging, requests, and other web things. This will make it eaiser to develop a web app by not recreating the wheel.
  • Python as a backend language of choice
    • By using python as our backend language instead of PHP, we can avoid mixing backend languages with the frontend HTML, thus allowing greater separation of concerns. Also, due to the wealth of debugging tools available for Python, development is significantly easier than it is with PHP.
  • PostgreSQL as database and Heroku as hosting
    • Heroku allows free web hosting for small projects. It also allows one free PostgreSQL database. Since these are provided for us and make the deployment process easier than copying files to the university servers, we chose to use them.

Data

We will compare student seller prices against those from Google Books, Barnes & Noble, and Amazon, so the buyer can make an informed decision about buying. We will be getting most of our price comparison data directly from the websites of Google Books, Amazon, and Barnes & Noble. Student seller prices will be displayed next to the aforementioned competitors, allowing for a better informed decision about book purchases. We have already began with some development of an automatic webscraper, written in python, to search multiple book websites, retrieving data from the search pages of those book selling sites.
